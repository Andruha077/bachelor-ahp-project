# Інструкція з оновлення проєкту

Даний документ описує процедуру оновлення веб-застосунку моделювання системи вибору методом аналізу ієрархій (МАІ) у production середовищі.


# 1. Підготовка до оновлення

Перед початком оновлення необхідно:

1. Перевірити наявність нової версії коду у репозиторії.
2. Ознайомитися зі змінами у новій версії (commit history або release notes).
3. Переконатися, що сервер працює стабільно.


# 2. Створення резервних копій

Перед оновленням необхідно створити резервну копію поточної версії сайту.

Приклад команди:
sudo cp -r /var/www/html /var/www/html_backup

Це дозволить швидко відновити попередню версію у випадку помилки.


# 3. Перевірка сумісності

Перед розгортанням нової версії необхідно перевірити:

- чи не змінено структуру файлів
- чи не додано нові залежності
- чи не змінився основний файл запуску

Оскільки проєкт є статичним веб-застосунком, сумісність перевіряється шляхом тестового запуску нової версії у браузері.


# 4. Планування часу простою

У більшості випадків оновлення статичного сайту займає кілька секунд.

Тому час простою системи мінімальний і може виконуватися без попереднього повідомлення користувачів.



# 5. Процес оновлення

## 5.1 Зупинка необхідних служб

Для статичного сайту зупинка сервера зазвичай не потрібна.

За необхідності можна тимчасово зупинити веб-сервер:
sudo systemctl stop nginx


## 5.2 Розгортання нового коду

Оновлення коду виконується за допомогою Git.

Перейти у папку з проектом:
cd /var/www/html

Отримати нову версію:
git pull

Якщо проект копіюється вручну, можна просто замінити файл:
sudo cp index.html /var/www/html/

## 5.3 Міграція даних

Даний проєкт не використовує базу даних, тому міграція даних не потрібна.


## 5.4 Оновлення конфігурацій

Якщо у новій версії змінено конфігурацію веб-сервера, необхідно оновити файл конфігурації Nginx або Apache.

У поточній версії проєкту додаткових конфігурацій не використовується.


# 6. Перевірка після оновлення

Після завершення оновлення необхідно:

1. Перезапустити веб-сервер за потреби
   sudo systemctl restart nginx

   Відкрити сайт у браузері
   
   2. Перевірити:

- відкриття головної сторінки
- роботу інтерфейсу
- виконання обчислень методу МАІ
- коректне відображення результатів


# 7. Процедура відкату (Rollback)

У разі виникнення помилки після оновлення необхідно повернути попередню версію.

1. Видалити поточну версію:
   sudo rm -rf /var/www/html/*
   
2. Та відновити резервну копію:
  sudo cp -r /var/www/html_backup/* /var/www/html/

3. Перезапустити веб-сервер:
   sudo systemctl restart nginx
   
   Після цього сайт буде працювати на попередній стабільній версії.
