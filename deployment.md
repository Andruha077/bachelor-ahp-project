# Інструкція з розгортання проєкту у production середовищі

Даний документ описує процедуру розгортання веб-застосунку моделювання системи вибору методом МАІ у production середовищі.

---

# 1. Вимоги до апаратного забезпечення

Оскільки проєкт є статичним веб-застосунком, вимоги до сервера мінімальні.

Мінімальні характеристики сервера:

Архітектура: x86_64  
CPU: 1 ядро  
Оперативна пам'ять: 512 MB  
Дисковий простір: 100 MB  

Операційна система:

Linux (Ubuntu / Debian / CentOS)  
або Windows Server

---

# 2. Необхідне програмне забезпечення

Для роботи проєкту необхідно встановити:

- веб-сервер (Nginx або Apache)
- Git (для отримання коду з репозиторію)
- веб-браузер для перевірки роботи

Рекомендована конфігурація:

Nginx + Linux сервер.

---

# 3. Налаштування мережі

Необхідно відкрити стандартні HTTP порти:

Port 80 – для доступу до веб-сайту  
Port 443 – для HTTPS (за потреби)

Сервер повинен мати статичну IP-адресу або доменне ім’я.

---

# 4. Конфігурація серверу

Встановити веб-сервер Nginx.

Ubuntu / Debian:

sudo apt update
sudo apt install nginx

Після встановлення переконатися, що сервер запущений:

sudo systemctl status nginx

Файли сайту розміщуються у директорії:

/var/www/html/

---

# 5. Налаштування СУБД

Даний проєкт **не використовує базу даних**, оскільки всі обчислення виконуються на стороні клієнта (JavaScript).

Тому встановлення СУБД не потрібне.

---

# 6. Розгортання коду

1. Клонувати репозиторій:

git clone https://github.com/username/project-name

2. Перейти у папку проєкту:

cd project-name

3. Скопіювати файли на сервер:

sudo cp index.html /var/www/html/

або скопіювати весь проєкт:

sudo cp -r * /var/www/html/

4. Перезапустити веб-сервер:

sudo systemctl restart nginx

---

# 7. Перевірка працездатності

Щоб перевірити чи проєкт працює:

1. Відкрити браузер
2. Перейти за адресою сервера

http://server-ip

або

http://domain-name

Якщо сторінка з інтерфейсом системи МАІ відкривається та працює обчислення, то розгортання виконано успішно.

---

# Висновок

Після виконання всіх кроків веб-застосунок буде доступний для користувачів через браузер та готовий до використання у production середовищі.
